Sensor Menu
 

Configure
 

 

The configuration of the parameters of the sensor have many options:

Sensor configuration

Regarding the sensor selected in you document, you have one of those tabs:
 

HCFR Sensor
 

This screen lets you define the configuration of colorimetric sensor HCFR.

The configuration frame makes it possible to define the parameters of communication between PC and the sensor.


 
 

Configuration

Port Com / USB  
Allows to define the port COM on which the sensor is connected. Sensor HCFR uses a port USB in emulation of port series. It is possible to manually select the port COM concerned (if it is known), but it is much more practical to let HCFR Colorimeter detect the sensor automatically by leaving the choice by defect: USB.


Timeout Mesure 
Indicate the number of milliseconds before a timeout. The default value of 40000 is sufficient in the majority of the cases, but sometimes, in particular during measurements of very weak luminosities (longer to measure) by requiring a strong precision (interlacing, use of the two sensors…), it can become necessary to increase this value to prevent that HCFR Colorimeter does not give an error of timeout during a measurement.

Debug  
This function makes it possible to save a file of log of the measurements returned by sensor HCFR in native format. It is not useful to generate this file of normal use.

Version  
The button “Version” makes it possible to question sensor HCFR and to post the version of its internal firmware. That also makes it possible to validate the correct operation of connection between the PC and the sensor.

 

Advances prameters

The framework of the advanced parameters makes it possible to define the parameters of operation of sensor HCFR during a measurement.

Sensor used
Allows to define which of two sensors TAOS embarked on the probe must be employed. It is possible to make measurements on the two sensors (what doubles the time of measurement) and to make the average of the two results to obtain more precision.

Interlacing of measurements
Allows to multiply the number of successive measurements carried out by the probe and to make the average of it, in order to obtain a larger precision of measurements. That multiplies the time of measurement of the same factor as interlacing.

Measure the white
Allows to measure the white in addition to the three components red, green and blue. This value not being currently exploited by HCFR Colorimeter, it should be left not notched, because it lengthens times of measurement unnecessarily.

Fast Measure
This option divides by two the unit time of measurement. It is very practical “to quickly trim” a diffuser, It is possible to jointly use it with measurements on two sensors and/or interlaced measurements, but as these options are paradoxical, it is preferable to only use it.

 

 

Simulated sensor

This box makes it possible to define the various error rates to the measures returned by the simulated sensor. If the three offsets to the zero and three checkbox are not notched, the simulated sensor will return exactly the color required by  HCFR Colorimeter.



 



Spyder II

This box lets you define initialization parameters for Spyder II.


 

The calibration mode can be defined on Default, CRT, LCD or PROJ. We have few information concerning the differences between these modes. The experiment shows that mode CRT is the best for the diffusers of the type CRT, and mode LCD is good for all the other types of diffusers (mode selected by defect). The other modes are to be regarded as experimental.

Read time (ms): This value is transmitted to Spyder II at the time of a request for measurement. It makes it possible to influence over the time devoted to a measurement. For a highher degree of accuracy, ColorHCFR adjusts this value so that it is higher in the dark nuances. The time of measurement is thus appreciably longer for the values IRE < 30 and the colors primary and secondary.

Extend read time on dark measurements of measurement for the dark colors: When this box is notched, the time of measurement is increased for the gray dark, the primary and secondary colors, in order to increase the precision.

Average many reads on dark measurements: When this box is notched, several successive measurements are taken until reaching a preset threshold of brightness (with a maximum number of authorized measurements), and the software calculates an average of all the results obtained. This option lengthens the time of measurement for the dark colors, but makes it possible to increase the precision of the results significantly.

 

DTP-94

This box lets you define initialization parameters for DTP-94.

 

The calibration mode can be defined on Default, CRT, or LCD. Mode CRT is the best for the diffusers of the type CRT, and mode LCD is good for all the other types of diffusers (mode selected by defect).

Timeout (ms): timeout of communication. This time does not have any influence to the measure, it is only a timeout related to connection USB. There is no particular reason to improve it.

Average many reads on dark measurements: When this box is notched, several successive measurements are taken until reaching a preset threshold of brightness (with a maximum number of authorized measurements), and the software calculates an average of all the results obtained. This option lengthens the time of measurement for the dark colors, but makes it possible to increase the precision of the results significantly.

Black level: The check box “Use internal device black level offsets” makes it possible to withdraw of all measurements of constant the correspondent the values returned by the internal sensors in the presence of an absolute black (internal calibration of the probe). This checkbox is notched by defect, and must be stripped only on an experimental basis, because that reduces the precision of measurements.

The button “Calibrate internal sensor offsets ” makes it possible to take a measurement of the black, and to store the values in the memory flash of the DTP-94, in order to withdraw them measurements to come. To take this measurement, the sensor should be sealed completely so that it measures an absolute black. Measurement takes approximately 8 seconds, and the measured values are posted. It is normally not necessary to proceed to this operation.

The button “Measure internal device temperature” require the DTP-94 to return its information of internal temperature. The interest of this function is purely experimental.

The check box “Debug..” makes it possible to store the whole of the dialog with the DTP-94 in the decoder.log file. It is not useful to generate this file of normal use.

 

 

Eye-One

This box lets you define initialization parameters for Eye One (Display model or Pro).  



The calibration mode can be defined on Eye One Display - CRT, Eye One Display - LCD, Eye One Display - Plasma, and Eye One Pro. Mode CRT is the best for the diffusers of the type CRT, and mode LCD is good for all the other types of diffusers (mode selected by defect). Plasma mode is reserved to plasma screens. This mode is a CRT mode without synchronisation on refresh frequency. Eye One Pro mode is reserved to Eye One Pro device.

Mode "ambient light" (using light diffusor): this mode is accessible only for Eye One Display. It changes slightly results and can be used when performing measures facing projector, using the light diffusor. Important note : when this mode is selected, the diffusor set on the probe, and the probe is facing projector, measured Y values are in Lux. In this mode, the Eye One Display can be used as a luxmeter.

Average many reads on dark measurements: When this box is notched, several successive measurements are taken until reaching a preset threshold of brightness (with a maximum number of authorized measurements), and the software calculates an average of all the results obtained. This option lengthens the time of measurement for the dark colors, but makes it possible to increase the precision of the results significantly.

The button “Calibrate internal sensor offsets ”  makes it possible to carry out calibration offsets intern of the probe, in order to obtain reliable measurements. This calibration must be carried out very regularly to claim to obtain precise figures. Note: In CRT mode, probe calibration is made by measuring a white surface displayed by the CRT to calibrate. This way, the probe can synchronize itself on CRT frequency. In all other modes, calibration consists in measuring an absolute black (light input closed).

Debug: This function makes it possible to save a file of log the whole of the measurements returned by Eye One (including the spectral data for Eye One Pro). It is not useful to generate this file of normal use.

 

Mazet MTCS-C2

This box lets you define initialization parameters for Mazet MTCS-C2 probe.

 

Read time (ms): This value is transmitted to the probe at the time of a request for measurement. It makes it possible to influence over the time devoted to a measurement.

Extend read time on dark measurements of measurement for the dark colors: When this box is notched, the time of measurement is increased for the gray dark, the primary and secondary colors, in order to increase the precision.

Debug: This function makes it possible to save a file of log the whole of the measurements. It is not useful to generate this file of normal use.

Device internal conversion matrix: This zone allows internal sensor matrix handling. The matrix can be seen, edited, or written. You can also write a default matrix with reasonable precision, if you accidentally deleted the probe matrix. This function must be used with extreme care, if you don't want to make the probe unusable. To define precisely conversion matrix, you must have a reference colorimeter. First, input manually an identity matrix (values 1 in cases top/left, center and bottom/right, 0 in all other cases). Then, perform simultaneous measures of primary colors and white on Mazet probe with internal identity matrix, and the reference colorimeter. Then, compute XYZ adjustment matrix on Mazet probe document, regarding reference colorimeter document. Retrieve XYZ adjustment matrix and copy/paste it in thix box before writing it inside the probe eeprom.

Device internal black level: This zone lets you see and measure sensor reference black level. Black level calibration is very important for accurate measures, so it must be refreshed from time to time. To do so, click "Measure black level" to measure an absolute black with the probe (blocking light guide input, or for example putting the probe inside a black bag). This measure displays the values in edit boxes, and you can write those values inside the eeprom using the ad hoc button. Note: it can happen that measured values are out of range, and need an internal offsets calibration. In this case, a message box appears to indicate this situation, and you can perform a full offsets + black level calibration operation.

Sensor Matrix configuration

This box displays the conversion matrix between the triplets of sensor measurements and colors with format XYZ. This matrix comes from the calibration of the colorimetric sensor relative with the diffuser to be measured. It is possible to modify manually this matrix, but that is not of any interest if we dos not have necessary knowledge.

This matrix concerns only probes needing calibration files, ie HCFR probe and simulated sensor. It can also concern tri-stimulus probes if the "Activate calibration files on all tri-stimulus probes" is checked in advanced preferences.
 

The dialog displays the matrix creation date, the calibration file name, additivity information and author signature, if calibration file has been saved by version 2.0 or above.

 


 

 

 

Calibration

This box appears when one asks for a calibration of the sensor during the process of creation of a new document, or by calling  Sensor/Configure... . It allows to define colorimetric constants, from which procedure from calibration will be able to calculate the Matrix Sensor defining a File Profile. This screen is particularly useful when one does not have a standard available for diffuser to be gauged, but when his colorimetric characteristics are known for a given adjustment (for example at exit of paperboard, all adjustments with the neutral). It is then enough to enter the chromacities of the white and the primary colors that it diffuser will post during the calibration, and after a measurement of these colors by the sensor, the application will be able to calculate the matrix of corresponding conversion. This profile will be  used during measurements on this diffuser.

Options
 

During the calibration, a certain number of options make it possible to control or influence the calculation of the matrix of conversion.

Black Compensation
This option indicates that the values measured for each component at the black level it must be withdrawn values measured on the primary colors. That makes it possible to be abstracted from the parasitic lights emitted by the diffuser in the black, and thus to obtain a matrix of more precise conversion. It is notched by defect.

Check additivity
This option does not affect the calculation of the matrix of conversion. It makes it possible to check the coherence of measurements between the primary colors and the white. It is checked that the white is well the sum of the red, the green and blue. A certain percentage of error (skeletal) is tolerated, but beyond this percentage, the calibration will be refused.

Display additivity results
This option makes it possible to post the variations of additivity on each component after the calibration.

IRE  level images
This option makes it possible to modify the IRE level of the tests pattern emitted by the generator “DVD manual” during the calibration. It is recommended to leave it with the default value of 100 IRE.
