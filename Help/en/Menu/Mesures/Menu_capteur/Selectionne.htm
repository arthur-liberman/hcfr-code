HCFR Colorimeter

Sensor Menu

 

Select

This dialog box appears during the process of creating of a new file of measurement, or when one calls small Sensor/Select. It makes it possible to choose among the colorimetric sensors available, and to calibrate these sensors.
 


 

 The dialog box inform you that your current parameters of the sensor in  use will be lost.
 


 

Valid (Yes).

 

The dialog box "Select sensor" is available. 
 

 

There are currently four managed sensors.

HCFR sensor :

It is the HCFR colorimetric sensor.
 

Simulated sensor:

It is not a genuine sensor, but simply a routine making it possible to return a simulated measurement of color. It is obviously not possible to directly carry out the least calibration with this type of sensor, but it makes it possible all the same to use the application HCFR Colorimeter alone. That can be useful for those which have a colorimeter not managed by HCFR Colorimeter, and which wishes to seize measurements manually in order to have the functions of calculation and posting of the application.

In this dialog box, it should be specified how the sensor must be calibrated:

To use a file of existing calibration:

Select the profile corresponding to the diffuser to gauge among those available in the combo box.

To launch a new calibration of the sensor:

The profile will be carried out by a measurement of the primary colors, white, and black. This procedure makes it possible to calibrate the sensor for a diffuser which one knows the gauged beforehand. If one has neither a standard, nor of a gauged projector, HCFR Colorimeter will not be able to ensure the exactitude of colorimetric measurements around the D65 point. However, that does not remove any interest with the application, because other measurements remain coherent and exploitable.

A profile carried out by this procedure can be saved in a file of extension .thc and be re-used thereafter.

 

The calibration relates only to sensor HCFR and the simulated sensor. Sensor Spyder II , DTP-94 and Eye-one do not need this standard. The corresponding zones are not available when one of these sensors is used.
