Option"Blanking"

This box determines the parameters valid for all the generators. There is only one option available:

 

   
Blank screen during measurement:

This option relates to the PC monitor used for the calibration. Indeed, to prevent that the screen of the monitor does not disturb measurements by adding parasitic light in the part, HCFR Colorimeter posts a black screen.

 

HCFR Generator

This generator is integrated into sensor HCFR. This generator orders in a semi-automatic way a reader of DVD via an infra-red lamp able to emulate the actions of the remote control. This generator must be used in correlation with the DVD of test cards HCFR.
 

 

This box makes it possible to define the parameters of the generator IR integrated into the probe colorimetric HCFR.
 

  • Display Dialog :
    This check box makes it possible to make post a box of message at the beginning of each series of test cards asked. Measurement starts only when one clicks on OK.
     

  • COM Port:
    Allows to define the port COM on which the probe is connected. Probe HCFR uses a port USB in emulation of port series. It is possible to select manually the port COM concerned (if it is known), but it is much more practical to leave ColorHCFR to detect the probe automatically by leaving the choice by defect: USB.

 

  • IR Profile
    You can choose an existing IR profile, delete an old one or create new profiles. The New and Modify buttons open the IR profile definition box.
     

  • Pattern change check
    This group activates and configures the pattern change checker. This function checks at each measure if the pattern really changed by comparison with the previous measure. If the pattern is identical, the IR code is re-emitted, and the measure is restarted.

    -     Minimal threshold : Check sensitivity setting. Higher this value, higher the chances to detect the pattern change, but the risk of bad detection increases too.

    -     Maximum retry number by pattern : Define the max number of retry measures on each pattern after first try.

    -     Maximum retry number by series : Define the max number of retry measures for each measure series. This number must be higher than maximum number by pattern.

    -     Activate on : Define on which series the checker applies. It is not possible to use this feature on near black and near white gray scales, because the difference is too thin between two consecutive patterns.

 

 

Boîte de définition des profils IR

Cette boîte permet de définir les codes IR correspondant à un modèle de lecteur de DVD particulier.

 

 

  • Nom
    Cette zone permet de saisir le nom du profil (nom du fichier contenant le profil IR)

  • Code IR pour chapitre suivant :
    Il est possible de choisir parmi 4 possibilités.
    1 Chapitre suivant
    2 OK
    3 Flèche en bas
    4 Flèche en haut
    Selon le choix ci-dessus, la zone de saisie contient le code IR correspondant, exprimé sous forme de nombres en hexadécimal. Ce code peut être saisi manuellement à partir d’une base de référence, ou acquis en utilisant la sonde ColorHCFR.

    Pour utiliser ce générateur, seul le code « Chapitre suivant » est absolument nécessaire. S’il est le seul à être renseigné, le Colorimètre HCFR demandera de se positionner manuellement sur la première mire de la série requise, et passera d’une mire à l’autre en émettant ce code.

    Lorsque les quatre codes sont renseignés, le Colorimètre HCFR exigera simplement d’être sur le menu principal avec la première option sélectionnée. Il pourra alors automatiquement naviguer dans le menu pour sélectionner la bonne série de mires. Il n’est alors plus nécessaire de naviguer manuellement entre les séries de mires, sauf dans le cas de mesures avancées (saturations par exemple).

  • Test de validité:
    Ce bouton permet de tester la validité syntaxique du code saisi. Cela ne garantit pas que le code fonctionnera avec le lecteur de DVD source, simplement que le code est au bon format. Si le code n’est pas correct syntaxiquement, il ne sera pas enregistré.

  • Acquérir le code :
    Ce bouton lance l’acquisition du code IR par la sonde HCFR. La sonde HCFR doit être branchée au PC. Lorsqu’on clique sur ce bouton, on a 10 secondes pour procéder à l’acquisition. La LED verte de la sonde s’éteint pendant l’acquisition. Avec la télécommande du lecteur de DVD, appuyer sur la touche « chapitre suivant », « OK », « Flèche en bas » ou « Flèche à droite » en visant le capteur colorimétrique le plus proche de la LED verte. Dès que l’acquisition est réalisée, le code apparaît dans la zone de saisie. Attention à ce que les codes acquis correspondent bien à la touche sélectionnée dans la combobox.

    Le capteur n’étant pas conçu à la base pour recevoir des infrarouges, sa sensibilité est relativement faible, et il est facilement perturbé par la lumière ambiante, pour laquelle il est au contraire très sensible. Il faut donc que la télécommande soit proche du colorimètre (quelques centimètres), et qu’il fasse aussi sombre que possible dans la pièce au moment de l’acquisition.

  • Envoi du code :
    Ce bouton permet de faire émettre immédiatement code IR par la sonde HCFR. Il est ainsi possible de vérifier que le lecteur de DVD réagit correctement au code, ce qui prouve indubitablement qu’il a été correctement acquis.

  • Temps de latence :
    Les temps de latence permettent de déterminer le nombre de millisecondes à attendre entre deux commandes de navigation de menu (les appuis de touches flèches pour déplacer le curseur à l’intérieur d’un menu), le temps de validation menu (après avoir envoyé le code OK, il faut patienter un peu avant l’apparition de la première mire), et le temps d’attente après avoir envoyé le code « chapitre suivant ».

    Attention : si votre lecteur de DVD affiche des informations en surimpression sur les mires quand on passe d’un chapitre à l’autre, il faut en tenir compte et augmenter le temps de latence. En effet, si les mesures démarrent avant que ces informations aient disparu, les résultats seront faussés.

    Note     : si les codes correspondant aux touches OK, flèche vers le bas et flèche vers la droite ne sont pas saisis, les temps de latence de navigation menu et de validation menu sont sans effet.
