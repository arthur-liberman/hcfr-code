Configure
 

 

La configuration des paramètres du capteur (simulé, Spyder II, DTP-94  ou Colorimètre HCFR ) comporte plusieurs onglets :

Configuration du capteur

Suivant l'option choisi dans le menu capteur/sélectionne vous avez:
soit la "Sonde HCFR"
soit le capteur simulé

Cas de la sonde HCFR
 

Cet écran permet de définir la configuration de la sonde colorimétrique HCFR.
Le cadre Configuration permet de définir les paramètres de communication entre le PC et la sonde.


 
 

Configuration

Port Com / USB  
Permet de définir le port COM sur lequel la sonde est branchée. La sonde HCFR utilise un port USB en émulation de port série. Il est possible de sélectionner manuellement le port COM concerné (si on le connaît), mais il est beaucoup plus pratique de laisser Colorimètre HCFR détecter automatiquement la sonde en laissant le choix par défaut : USB.


Timeout Mesure 
Indique le nombre de millisecondes avant un timeout. La valeur par défaut de 40000 est suffisante dans la plupart des cas, mais parfois, notamment lors de mesures de très faibles luminosités (plus longues à mesurer) en exigeant une forte précision (entrelacement, usage des deux capteurs…), il peut devenir nécessaire d’augmenter cette valeur pour éviter que Colorimètre HCFR ne donne une erreur de timeout lors d’une mesure.

Debug  
Cette fonction permet de sauver un fichier de log des mesures renvoyées par la sonde HCFR en format natif. Il n’est pas utile de générer ce fichier en utilisation normale.

Version  
Le bouton « Version » permet d’interroger la sonde HCFR et d’afficher la version de son firmware interne. Cela permet également de valider le bon fonctionnement de la connexion entre le PC et la sonde.

 

Paramètres avancés

Le cadre des paramètres avancés permet de définir les paramètres de fonctionnement de la sonde HCFR lors d’une mesure.

Capteur utilisé
Permet de définir lequel des deux capteurs TAOS embarqués sur la sonde doit être employé. Il est possible de faire les mesures sur les deux capteurs (ce qui double le temps de mesure) et de faire la moyenne des deux résultats pour obtenir plus de précision.

Entrelacement des mesures
Permet de multiplier le nombre de mesures successives réalisées par la sonde et d’en faire la moyenne, afin d’obtenir une précision plus grande des mesures. Cela multiplie le temps de mesure du même facteur que l’entrelacement.

Mesure le blanc
Permet de mesurer le blanc en plus des trois composantes rouge, verte et bleue. Cette valeur n’étant pas actuellement exploitée par Colorimètre HCFR, il faut la laisser non cochée, car elle allonge inutilement les temps de mesure.

Mesure rapide
Cette option divise par deux le temps de mesure unitaire. Elle est très pratique pour « dégrossir » rapidement un diffuseur, Il est  possible de l’utiliser conjointement avec les mesures sur deux capteurs et/ou avec les mesures entrelacées, mais comme ces options sont antinomiques, il est préférable de l’utiliser seule.
 

Cas du capteur simulé

Cette boîte permet de définir les différents taux d’erreur sur les mesures renvoyées par le capteur simulé. Si les trois offsets sont à zéro et les trois checkbox non cochées, le capteur simulé renverra exactement la couleur demandée par Colorimètre HCFR.



 



Cas du Spyder II

Cette boîte permet de définir les paramètres d’initialisation du Spyder II.


 

Le mode de calibrage peut être défini sur Défaut, CRT ou LCD. Nous avons peu d’informations concernant les différences entre ces modes. L’expérience montre que le mode CRT est le meilleur pour les diffuseurs de type CRT, et le mode LCD est bon pour tous les autres types de diffuseurs (mode sélectionné par défaut). L'autre mode est à considérer comme expérimental.

Temps de mesure (ms) : Cette valeur est transmise au Spyder II lors d’une demande de mesure. Elle permet d’influer sur le temps consacré à une mesure. Pour une plus grande précision, ColorHCFR ajuste cette valeur pour qu’elle soit plus élevée dans les nuances sombres. Le temps de mesure est donc sensiblement plus long pour les valeurs IRE < 30 et les couleurs primaires et secondaires.

Allonger le temps de mesure pour les couleurs sombres : Quand cette case est cochée, le temps de mesure est augmenté pour les gris sombres, les couleurs primaires et secondaires, afin d’augmenter la précision.

Moyenner plusieurs mesures pour les couleurs sombres : Quand cette case est cochée, plusieurs mesures successives sont effectuées jusqu’à atteindre un seuil de luminance prédéfini (avec un nombre maximum de mesures autorisées), et le logiciel calcule une moyenne de tous les résultats obtenus. Cette option allonge le temps de mesure pour les couleurs sombres, mais permet d’augmenter significativement la précision des résultats.

Cas du DTP-94

Cette boîte permet de définir les paramètres d’initialisation du DTP-94.

Le mode de calibrage peut être défini sur Défaut, CRT, ou LCD. Le mode CRT est le meilleur pour les diffuseurs de type CRT, et le mode LCD est bon pour tous les autres types de diffuseurs (mode sélectionné par défaut).

Timeout (ms) : timeout de communication. Ce temps n’a aucune influence sur la mesure, c’est uniquement un timeout lié à la liaison USB. Il n’y a pas de raison particulière de le retoucher.

Moyenner plusieurs mesures pour les couleurs sombres : Quand cette case est cochée, plusieurs mesures successives sont effectuées jusqu’à atteindre un seuil de luminance prédéfini (avec un nombre maximum de mesures autorisées), et le logiciel calcule une moyenne de tous les résultats obtenus. Cette option allonge le temps de mesure pour les couleurs sombres, mais permet d’augmenter significativement la précision des résultats.

Niveau de noir :

La check box « Utiliser les offsets de niveau de noir de la sonde » permet de soustraire de toutes les mesures des constantes correspondant aux valeurs renvoyées par les capteurs internes en présence d’un noir absolu (étalonnage interne de la sonde). Cette checkbox est cochée par défaut, et ne doit être décochée qu’à titre expérimental, car cela réduit la précision des mesures.

Le bouton « Calibrer les offsets internes de la sonde » permet d’effectuer une mesure du noir, et de stocker les valeurs dans la mémoire flash du DTP-94, afin de les soustraire des mesures à venir. Pour effectuer cette mesure, il faut obturer totalement le capteur pour qu’il mesure un noir absolu. La mesure prend environ 8 secondes, et les valeurs mesurées sont affichées. Il n’est normalement pas nécessaire de procéder à cette opération.

Le bouton « Mesurer la température interne de la sonde » permet de demander au DTP-94 de renvoyer son information de température interne. L’intérêt de cette fonction est purement expérimental.

La check box « Debug » permet de stocker l’ensemble du dialogue avec le DTP-94 dans le fichier decoder.log. Il n’est pas utile de générer ce fichier en usage normal.

 

Cas du Eye-One

Cette boîte permet de définir les paramètres d’initialisation du Eye One (modèle Display ou Pro).



Le mode de calibrage peut être défini sur Eye One Display - CRT, Eye One Display - LCD, Eye One Display - Plasma, et Eye One Pro. Les trois premiers modes doivent être utilisés pour la sonde Eye One Display 2 ou Eye One Display LT, en fonction du type de diffuseur à mesurer. Le mode "CRT" est à employer pour les écrans cathodiques et les projecteurs tritubes. Le mode "LCD" est à employer pour les écrans plats LCD, et les projecteurs LCD, DLP et DILA/SXRD. Le mode Plasma est réservé aux écrans plasma. Ce dernier mode est en fait un mode CRT sans synchronisation sur une fréquence de balayage particulière. Le mode Eye One Pro est réservé au spectrophotomètre Eye One Pro, qui n'a pas besoin de connaître le type de diffuseur à étalonner.

Mode "lumière ambiante" (utilisation du diffuseur): ce mode n'est accessible que pour le Eye One Display. Il change légèrement les résultats, et peut être utilisé lorsqu'on effectue des mesures face au projecteur en utilisant le diffuseur de lumière. Note importante: lorsque ce mode est sélectionné, que le diffuseur est monté sur la sonde, et qu'on effectue des mesures face au projecteur, les valeurs de Y renvoyées sont en Lux. Dans ce mode, le Eye One Display peut donc être utilisé comme un luxmètre.

Moyenner plusieurs mesures pour les couleurs sombres : Quand cette case est cochée, plusieurs mesures successives sont effectuées jusqu’à atteindre un seuil de luminance prédéfini (avec un nombre maximum de mesures autorisées), et le logiciel calcule une moyenne de tous les résultats obtenus. Cette option allonge le temps de mesure pour les couleurs sombres, mais permet d’augmenter significativement la précision des résultats.

Calibrer les offsets internes de la sonde : Ce bouton permet de procéder au calibrage des offsets interne de la sonde, afin d’obtenir des mesures fiables. Ce calibrage doit être effectué très régulièrement pour prétendre obtenir des chiffres précis. Note: En mode Eye One Display - CRT, le calibrage de la sonde doit être fait sur l'écran CRT à étalonner, par mesure d'une surface blanche. Ceci permet à la sonde de se synchroniser sur la fréquence de balayage du CRT. Dans tous les autres modes, il s'agit de mesurer un noir absolu (en obturant l'entrée de lumière).

Debug : Cette fonction permet de sauver un fichier de log l’ensemble des mesures renvoyées par le Eye One (y compris les données spectrales pour le Eye One Pro). Il n’est pas utile de générer ce fichier en utilisation normale.

 

Cas de la sonde Mazet MTCS-C2

Cette boîte permet de définir les paramètres d’initialisation de la sonde Mazet MTCS-C2.


 

Temps de mesure (ms) : Cette valeur permet d’influer sur le temps consacré à une mesure.

Allonger le temps de mesure pour les couleurs sombres : Quand cette case est cochée, le temps de mesure est augmenté pour les gris sombres, les couleurs primaires et secondaires, afin d’augmenter la précision.

Debug : Cette fonction permet de sauver un fichier de log l’ensemble des mesures renvoyées par la sonde. Il n’est pas utile de générer ce fichier en utilisation normale.

Matrice de conversion interne du capteur: Cette zone permet de gérer la matrice interne du capteur. Cette matrice peut être visualisée, éditée, et réécrite. On peut également écrire une matrice par défaut ayant une précision raisonnable, en cas d'effacement par erreur de la matrice du capteur. Cette fonction doit être utilisée avec la plus extrême précaution, sous peine de rendre la sonde inutilisable. Pour définir avec précision la matrice interne de la sonde, il faut disposer d'un colorimètre de référence. On entre manuellement dans les cases une matrice identité (valeurs 1 dans les cases haut/gauche, centrale et bas/droite, et des zéros partout ailleurs). On effectue des mesures simultanées des couleurs primaires et du blanc avec la sonde Mazet ayant une matrice identité, et le colorimètre de référence, et on calcule la matrice d'ajustement XYZ. On récupère alors la matrice d'ajustement XYZ, et par un simple copier/coller, on peut la rentrer dans cette zone.

Niveau de noir interne du capteur: Cette zone permet de visualiser et de mesurer le niveau de noir du capteur. L'étalonnage du noir est important pour la précision des mesures, il doit donc être fait de temps en temps. Pour cela, on clique sur le bouton "Mesurer le niveau de noir", en ayant pris garde de mettre la sonde dans le noir le plus complet (en obturant le guide lumière, ou mieux en enfermant la sonde dans un sac noir). Cette mesure affiche les valeurs du noir, qu'il convient d'écrire dans l'eeprom à l'aide du bouton ad hoc. Note: il peut arriver que ces valeurs soient hors limites, et nécessitent un étalonnage des offsets internes en plus du niveau de noir. Dans ce cas, un message l'indique, et le calcul des offsets de base et du niveau de noir peut être effectué.

Configuration de la Matrice du capteur

Cette boîte affiche la matrice de conversion entre les triplets de mesures capteur et les couleurs au format XYZ. Cette matrice provient de l’étalonnage du capteur colorimétrique relativement au diffuseur à mesurer. Il est possible de modifier manuellement cette matrice, mais cela ne présente aucun intérêt si on ne dispose pas des compétences nécessaires.

Cette matrice ne concerne que les sondes ayant besoin d'un étalon, donc la sonde HCFR et le capteur simulé. Cela peut aussi concerner les sondes tri-stimulus du commerce si l'option Active les étalons pour toutes les sondes tri-stimulus est activée dans les préférences avancées.
 

La boîte affiche également la date à laquelle la matrice de conversion a été créée, ainsi que le nom de l'étalon, et les informations d'additivité et de signature, si l'étalon a été sauvé par la version 2.0 ou ultérieure du logiciel.

 


 

 

 

Configuration de la calibration

Cette boîte apparaît lorsque on demande un étalonnage du capteur lors du processus de création d’un nouveau document, ou en appelant le menu Capteur / Configure. Elle permet de définir des constantes colorimétriques, à partir desquelles la procédure d’étalonnage va pouvoir calculer la Matrice Capteur définissant un Fichier Etalon. Cet écran est particulièrement utile quand on n’a pas d’étalon disponible pour le diffuseur à calibrer, mais quand on connaît ses caractéristiques colorimétriques pour un réglage donné (par exemple en sortie de carton, tous réglages au neutre). Il suffit alors d’entrer les chromacités du blanc et des couleurs primaires que le diffuseur va afficher lors de l’étalonnage, et après une mesure de ces couleurs par le capteur, l’application pourra calculer la matrice de conversion correspondante. Cet étalon sera a utiliser lors des mesures sur ce diffuseur.

 


 

Référence
 

Modèle  
Le modèle « Defaut » ne permet pas de saisir de chromacités, il propose celles du système de couleur de référence (accessible via le menu Préférences / Références ). Pour définir soi-même des chromacités, il faut sélectionner le modèle « Personnalisé »

Chromacité des primaires 
Cet écran est particulièrement utile quand on n’a pas d’étalon disponible pour le diffuseur à calibrer, mais quand on connaît ses caractéristiques colorimétriques pour un réglage donné (par exemple en sortie de carton, tous réglages au neutre). Il suffit alors d’entrer les chromacités des couleurs primaires que le diffuseur va afficher lors de l’étalonnage, et après une mesure de ces couleurs par le capteur, l’application pourra calculer la matrice de conversion correspondante. Cet étalon sera a utiliser lors des mesures sur ce diffuseur. 

Chromacité du blanc 
Cet écran est particulièrement utile quand on n’a pas d’étalon disponible pour le diffuseur à calibrer, mais quand on connaît ses caractéristiques colorimétriques pour un réglage donné (par exemple en sortie de carton, tous réglages au neutre). Il suffit alors d’entrer les chromacités du blanc que le diffuseur va afficher lors de l’étalonnage, et après une mesure de ces couleurs par le capteur, l’application pourra calculer la matrice de conversion correspondante. Cet étalon sera a utiliser lors des mesures sur ce diffuseur.

 

Options
 

Lors de l’étalonnage, un certain nombre d’options permettent de contrôler ou d’influencer le calcul de la matrice de conversion.

Compensation du noir
Cette option indique que les valeurs mesurées pour chaque composante au niveau du noir doivent être soustraites des valeurs mesurées sur les couleurs primaires. Cela permet de s’abstraire des lumières parasites émises par le diffuseur dans le noir, et ainsi d’obtenir une matrice de conversion plus précise. Elle est cochée par défaut.

Vérifie l'additivité
Cette option n’affecte pas le calcul de la matrice de conversion. Elle permet de vérifier la cohérence des mesures entre les couleurs primaires et le blanc. On vérifie que le blanc est bien la somme du rouge, du vert et du bleu. Un certain pourcentage d’erreur (paramétrable) est toléré, mais au delà de ce pourcentage, l’étalonnage sera refusé.

Affiche le résultat de l'additivité
Cette option permet d’afficher les écarts d’additivité sur chacune des composantes après l’étalonnage.

Niveau IRE des mires
Niveau IRE des mires :Cette option permet de modifier le niveau IRE des mires émises par le générateur « Affichage mires » pendant l’étalonnage. Il est recommandé de la laisser à la valeur par défaut de 100 IRE.
